§ 01

GPS Signal Structure

Every GPS satellite transmits a spread-spectrum signal on the L1 carrier at 1575.42 MHz. The signal is the product of three components: a PRN-unique C/A spreading code, a 50 bps navigation data bit stream, and the carrier. We generate each component independently, combine them, and stream the resulting complex baseband (I/Q) samples to the bladeRF's DAC. (IS-GPS-200L §3.3)

§ 02

C/A Gold Code Generator

Each GPS satellite uses a unique 1023-chip pseudorandom sequence — a Gold code — generated by XOR-ing the outputs of two 10-stage Linear Feedback Shift Registers (LFSRs). The generator polynomials and per-PRN phase offsets are defined in IS-GPS-200L Table 3-Ia.

The two generator polynomials over GF(2):
G1 POLYNOMIAL $$G_1(x) = x^{10} + x^3 + 1$$
G2 POLYNOMIAL $$G_2(x) = x^{10} + x^9 + x^8 + x^6 + x^3 + x^2 + 1$$
LFSR feedback equations — IS-GPS-200L §3.3.2
$$s_{11}^{(G1)} = s_3 \oplus s_{10}$$ $$s_{11}^{(G2)} = s_2 \oplus s_3 \oplus s_6 \oplus s_8 \oplus s_9 \oplus s_{10}$$
Both registers are initialised to all-ones: $s_1 \ldots s_{10} = 1$.
C/A chip output
$$CA_k[n] = G_1[n] \;\oplus\; G_2\!\bigl[n + \delta_k\bigr]$$
$\delta_k$ — G2 phase delay for PRN $k$, from IS-GPS-200L Table 3-Ia.
The delay is implemented by selecting specific output taps on G2 rather than physically advancing the register.

The key property of Gold codes is near-zero cross-correlation: $|R_{ij}(\tau)| \leq \frac{1+2^{(n+2)/2}}{2^n-1}$ for $n=10$, giving $\approx -24\;\text{dB}$. This allows 32 satellites to share the same 1575.42 MHz channel simultaneously. (Gold, 1967 — see §07)

§ 03

Navigation Message

The navigation message is broadcast at 50 bps. One complete frame is 1500 bits = 30 seconds. Each frame consists of 5 subframes × 10 words × 30 bits. Each bit spans exactly 20 C/A code repetitions (20 ms). (IS-GPS-200L §20.3)

§ 04

Satellite Position from RINEX Ephemeris

To generate a believable signal, we must compute where each satellite actually is in ECEF (Earth-Centered Earth-Fixed) coordinates at every GPS epoch. The orbital parameters are read from RINEX navigation files (updated daily by the IGS network) and solved via the IS-GPS-200 Keplerian algorithm. (IS-GPS-200L §20.3.3.4.3)

§ 05

Pseudorange & Doppler Shift

Given satellite ECEF position $\mathbf{r}_{sv}$ and spoofed receiver position $\mathbf{r}_u$, we compute the geometric range, code phase (pseudorange), and Doppler-shifted carrier frequency for each visible satellite independently. These three quantities drive the timing and frequency of each PRN channel in our signal generator. (Misra & Enge, 2006, Ch. 4)

Geometric range
$$\rho = \|\mathbf{r}_{sv} - \mathbf{r}_u\| = \sqrt{(x_{sv}-x_u)^2 + (y_{sv}-y_u)^2 + (z_{sv}-z_u)^2}$$
Code phase offset (chips)
$$\tau = \frac{\rho}{c} \cdot f_{chip} = \frac{\rho}{c} \times 1.023\times10^6$$
Carrier Doppler frequency
$$f_D = -\frac{(\mathbf{v}_{sv} - \mathbf{v}_u)\cdot\hat{\mathbf{e}}_{u\to sv}}{\lambda_{L1}}$$
Code Doppler (chip rate correction)
$$f_{chip,D} = f_D \cdot \frac{f_{chip}}{f_{L1}} \approx f_D \times 6.5\times10^{-4}$$
$\hat{\mathbf{e}}_{u\to sv}$ — unit vector from receiver to satellite
$\lambda_{L1} = c / f_{L1} \approx 0.1903$ m
For a stationary receiver: $\mathbf{v}_u = 0$, but satellite orbital velocity ($\approx 3.9$ km/s) still produces $\pm$5 kHz Doppler per channel.

For a stationary spoofed position, all dynamics come from actual satellite motion. The geometry changes every millisecond, so the signal generator recomputes $\rho$, $\tau$, and $f_D$ for each 1 ms epoch and updates code phase and numerically-controlled oscillator (NCO) frequency accordingly. For moving path mode (GPX track), $\mathbf{r}_u$ and $\mathbf{v}_u$ are interpolated from the track at each epoch.

§ 06

BPSK Baseband & I/Q

The combined code·data signal modulates the L1 carrier using Binary Phase Shift Keying (BPSK). Rather than synthesising an RF signal directly, we generate complex baseband I/Q samples — the bladeRF's DAC and built-in local oscillator perform the final upconversion to 1575.42 MHz. (Proakis, 2001, Ch. 5; Nuand bladeRF wiki)

Baseband I/Q per channel k (single PRN)
$$I_k(n) = A_k \cdot D_k(n) \cdot CA_k(n) \cdot \cos\!\left(2\pi f_{D,k}\frac{n}{f_s} + \varphi_k\right)$$ $$Q_k(n) = A_k \cdot D_k(n) \cdot CA_k(n) \cdot \sin\!\left(2\pi f_{D,k}\frac{n}{f_s} + \varphi_k\right)$$
Sum all visible satellites (N channels)
$$I(n) = \sum_{k=1}^{N} I_k(n) \qquad Q(n) = \sum_{k=1}^{N} Q_k(n)$$
Scale to bladeRF SC16Q11 format (Nuand bladeRF wiki)
$$I_{raw}(n) = \mathrm{round}\!\left(I(n)\cdot\frac{2047}{A_{max}}\right),\quad I_{raw} \in [-2048,\,2047]$$
$f_s$ — sample rate (2.048 – 61.44 Msps; use 2.6 Msps for GPS-only)
$A_{max}$ — peak amplitude after summing $N$ channels; scale to ~60% of full-scale to preserve headroom
Sample layout: [I₁, Q₁, I₂, Q₂, …] as interleaved int16_t pairs
